Quite Meticulous Logic

Overcoming QML's requirement of trivially constructable types

Michael Hasselmann

 Wed 21 August 2024
Category: Tips and Tricks
Tags: QML PySide6 Design Pattern

Introduction

In How objects are constructed and set up in QML we learned that types, or rather QML components, need to be trivially constructable. Apart from the implicit parent parameter, it is seemingly impossible for us to add more parameters to the hidden constructor. Instead, we rely on properties to set up the object's initial state.
